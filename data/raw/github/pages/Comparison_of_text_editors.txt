== Document interface ==
The following table identifies the following attributes of an editor that describe its user experience with respect to multiple documents:

Multiple instances
Whether multiple instances of the program can be open at the same time for editing multiple files; one in each instance. Applies both for single-document interface (SDI) and multiple-document interface (MDI). Also, applies for a program that has a user interface that looks like multiple instances of the same program (such as some versions of  Word).
Single document window splitting
Whether a window can be split to simultaneously view different areas of a file.
MDI overlappable windows
Whether each open document gets its own fully movable window inside the editor environment.
MDI tabbed document interface
Whether multiple documents can be viewed as tabs in a single window.
MDI window splitting
Whether supports splitting the application window to show multiple documents (non-overlapping windows).

== Programming features ==
A text editor that is designed for  programming may be classified as a source code editor. The following table describes an editor's support for the following programming features:

Syntax highlighting
Styles text (such as colors and fonts) according to syntactic rules.
Function list
Lists all functions from current file in a window or sidebar and allows user to jump directly to the definition of that function for example by double-clicking on the function name in the list. More or less realtime (does not require creating a symbol database, see below).
Symbol database
Database of functions, variable and type definitions, macro definitions etc. in all the files belonging to the software being developed. The database can be created by the editor itself or by an external program such as ctags. The database can be used to locate the definition even if it is in another file.
Bracket matching
Find matching start/end delimiters; taking into account nesting.
Auto indentation
May refer to just simple indenting to the same level as the line above, or intelligent indenting that is language specific, e.g., ensuring a given indent style.
 Auto completion
Suggesting the completion of a partially entered word such as a keyword or identifier
Code folding
Selectively show/hide blocks of code
Text folding
Selectively show/hide text on boundaries such as a paragraph or a section
Compiler integration
Allows running compilers/linkers/debuggers from within editor, capturing the compiler output and stepping through errors, automatically moving cursor to corresponding location in the source file.


== Additional programming features ==
This section identifies support for the following programming features:

CLI shell integration
Whether the editor integrates with a command-line interface (CLI) shell.
GUI shell integration
Whether the editor integrates with a graphical user interface (GUI) shell.
Macro language
Whether the editor supports a macro language for automation.
Collaborative editing
Whether the editor supports collaborative real-time editing.
Large file support
Most text editors do not support large text files. Some restrict themselves to available in-core RAM while others use sophisticated virtual memory management techniques and paging algorithms.

Long line support
Whether the editor supports long lines
Multi-line regex support
Whether the editor supports multi-line regex matching (e.g. "\n\n(.+)\n\n").
Search in files
Perform search and possibly replace in multiple files in the file system; for example on a sub-directory and recursively all the directories below it; like grep.
